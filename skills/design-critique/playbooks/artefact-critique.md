# Artefact Critique Playbook

Use this playbook for UX critique of static design artefacts.

## 1. Establish context

Capture what is known and what is assumed:

- Artefact type: screenshot, Figma frame, mockup, wireframe, exported image, or visual specification.
- Platform and viewport: desktop, mobile, tablet, kiosk, responsive breakpoint, or unknown.
- Audience: who the design is for, including relevant ability, context, expertise, or urgency factors.
- Primary objective: the main task or outcome the design should support.
- Design stage: early concept, wireframe, prototype mockup, pre-build UI, production review, or unknown.
- Critique depth: quick critique, full structured critique, visual accessibility focus, ecommerce/form focus, or objective-led critique.

If the missing context would materially change severity, ask one concise question. If the user wants speed, proceed with labelled assumptions.

## 2. Inspect the artefact

Review visible evidence:

- Information hierarchy and scan path.
- Primary and secondary calls to action.
- Navigation, orientation, and wayfinding.
- Form structure, labels, help text, and error-state evidence.
- Consistency of components, spacing, typography, labels, icons, and states.
- Accessibility-visible factors: apparent target size, target spacing, text contrast, non-text contrast, font size, and text density.
- Fit between design choices, audience, and primary objective.

## 3. Identify what works

Start with strengths worth preserving. Focus on strengths that support the objective, not generic praise.

## 4. Write severity-ranked findings

Use this format for each material issue:

- Issue: the visible problem in plain language.
- Principle: the source-backed UX or visual-accessibility principle involved.
- Consequence: what could happen to users or the business if it remains unchanged.
- Severity: Critical, High, Medium, Low, or Observation.
- Confidence: High, Medium, or Low.
- Recommendation: the smallest practical improvement that addresses the issue.

Prefer fewer, sharper findings over long undifferentiated lists. Combine duplicate issues under a single principle when the same root cause appears repeatedly.

## 5. Handle visual accessibility carefully

For target size, spacing, contrast, and font-size findings:

- Use exact measurements when dimensions or source colours are available.
- Use estimates when judging from screenshots or low-resolution exports.
- State when a finding is a likely risk rather than a conformance failure.
- Do not claim full accessibility compliance or failure unless the artefact provides enough data for that specific visible criterion.

## 6. Recommend next actions

End with the highest-impact fixes in priority order. Suggest additional research only when it would change the recommendation, such as competitor evidence, domain-specific norms, or user testing.
