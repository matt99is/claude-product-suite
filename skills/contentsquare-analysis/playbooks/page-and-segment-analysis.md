# Page And Segment Analysis

Use for page performance, mapping scope, device comparison, audience comparison,
and segment-led questions.

## Setup

1. Confirm the mapping and page group by object ID.
2. Define the metric family: traffic, engagement, progression, exit, conversion proxy, journey detour, or error exposure.
3. Select date range and comparison period.
4. Decide whether device, audience, or segment must be split.

## Validation

- Check whether several objects share the same name.
- Confirm segment size before reading performance.
- Do not compare segments with different entry denominators unless that is the point.
- Keep devices separate when layout or behaviour differs.
- Treat aggregate movement as weak when the affected component or journey is a small subset.

## Analysis

Start with what changed, then where it changed, then who it affected. Avoid
attributing movement to a launch or design change unless the release register and
surface scope support it.

## Output

Include:

- page or mapping scope;
- segment and device filters;
- headline read;
- evidence table or concise bullets;
- confidence and caveats;
- repeatability notes.
