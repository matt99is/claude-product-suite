# Stakeholder Summary

Use when packaging a Contentsquare analysis for a stakeholder summary, analyst
note, or report draft.

## Inputs

Confirm:

- question answered;
- data used;
- date range and comparison;
- device and segment filters;
- Contentsquare objects and object IDs;
- validation checks completed;
- MCP/UI caveats.

## Structure

1. Question answered.
2. Headline read.
3. Data used.
4. Findings.
5. Confidence and caveats.
6. Next checks.
7. Repeatability notes.

## Findings

Each finding should include:

- Observation:
- Interpretation:
- Recommendation:
- Next check:
- Caveat:

## Stakeholder Version

Keep it short. Use plain language. Include only the numbers that support the
decision. Avoid hype, urgency framing, and hidden method.

## Analyst Version

Include object IDs, validation checks, denominator notes, segment definitions,
and rerun steps. Make clear which evidence came from MCP and which requires UI
handoff.
