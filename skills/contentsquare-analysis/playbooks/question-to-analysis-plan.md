# Question To Analysis Plan

Use this playbook when the Contentsquare ask is vague, broad, or not yet tied to
a decision.

## Clarify The Ask

Capture:

- decision the analysis must inform;
- surface or journey area;
- audience, device, or segment;
- date range and comparison period;
- expected output;
- why the answer matters.

Ask one concise question if any of these would materially change the analysis.

## Load Context

Use `references/project-workspace-contract.md` to find the tracking plan, release
register, saved objects, working notes, and relevant prior reports.

## Choose Evidence

Map the question to the smallest Contentsquare evidence set:

- funnel or journey for step leakage;
- page or mapping for surface performance;
- segment or device comparison for population differences;
- zone analysis for component interaction;
- errors for technical friction;
- release register plus like-for-like comparison for launch impact.

## Plan Validation

State how you will validate object IDs, date range, device split, segment
population, denominator, optional steps, and MCP/UI capability before interpreting.

## Output

Return a short analysis plan with:

1. Question.
2. Data needed.
3. Contentsquare objects.
4. Validation checks.
5. Output format.
6. Open context needed.
