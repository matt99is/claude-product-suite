# Synthesize UserTesting.com Results

Use this playbook when the user supplies UserTesting.com results or asks how to export them for synthesis.

## 1. Ask for the right data

Request the UserTesting.com Excel export first when available. It is the preferred handoff because it may include transcripts, participant metadata, written responses, multiple choice answers, metrics, notes, and clip references.

If Excel export is unavailable, ask for the best available substitute:

- transcript exports;
- copied transcripts;
- clips or notes;
- metrics screenshots or tables;
- session-by-session notes;
- pasted observations.

Ask for participant labels, audience segments, task labels, and screener criteria when possible.

## 2. Inventory the data received

Before synthesizing, state what is present and what is missing:

- participant count;
- audience segments;
- tasks covered;
- transcripts or notes;
- metrics;
- quotes;
- clips or timestamps;
- limitations.

If the data is too thin for a confident synthesis, say so and offer a smaller output such as initial observations.

## 3. Code evidence lightly

Group evidence by:

- task;
- participant;
- theme;
- severity;
- frequency;
- audience segment, if relevant.

Keep coding practical. The goal is concise synthesis, not exhaustive qualitative analysis unless the user asks for it.

## 4. Write concise findings

For each finding, include:

- finding: one sentence;
- evidence: participant/task references and a short quote or behavior note when useful;
- impact: why it matters;
- recommendation: what to change or test next;
- confidence: high, medium, or low.

Separate observed behavior from analyst inference.

## 5. Add visuals where useful

Choose lightweight text-native visuals when they clarify the evidence:

### Participant x task matrix

| Participant | Task 1 | Task 2 | Task 3 | Notes |
|---|---|---|---|---|
| P1 | Pass | Struggled | Fail | Hesitated at pricing step |

### Severity x frequency table

| Theme | Severity | Participants | Evidence |
|---|---|---:|---|
| Pricing comparison unclear | High | 4/6 | P1, P2, P4, P6 |

### Theme map

| Theme | Related tasks | User impact | Recommended action |
|---|---|---|---|
| Navigation uncertainty | Task 1, Task 2 | Slower task completion | Rename category and expose entry point |

### Quote board

| Theme | Quote | Participant | Task |
|---|---|---|---|
| Trust | "I am not sure what happens next." | P3 | Task 2 |

Do not use visuals that imply statistical precision from a small qualitative sample.

## 6. Recommend next action

End with the smallest useful next step:

- design change;
- copy change;
- follow-up test;
- additional audience segment;
- analytics check;
- stakeholder decision.

## Output format

Use this order by default:

1. Study goal and data received.
2. Executive summary.
3. Top findings.
4. Evidence table.
5. Visual summary.
6. Recommended changes.
7. Confidence and caveats.
8. Suggested next test or follow-up.
