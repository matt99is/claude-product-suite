# Plan Unmoderated UserTesting.com Test

Use this playbook to create or review a concise UserTesting.com study.

## 1. Establish the brief

Capture the minimum useful brief:

- Decision to inform: what choice this research should support.
- Primary research goal: one clear goal.
- Target audience: who the test is for and why they are the right people.
- Relevant behavior or experience: what participants must have done or know.
- Device and platform: desktop, mobile, iOS, Android, browser, or prototype constraints.
- Artefact or experience: live site, prototype, concept, copy, flow, or competitor experience.
- Success criteria: what the team needs to observe or learn.
- Constraints: timing, sample size, geography, language, sensitivity, or launch deadline.
- What not to test: areas outside scope.

If the audience is missing, ask for it before drafting the script. If the user is unsure, propose a behavior-based audience and ask them to confirm.

## 2. Build the audience and screener plan

Define:

- target audience;
- inclusion criteria;
- exclusion criteria;
- segment groups, if any;
- device or platform constraints;
- screener questions.

Write screeners so the desired answer is not obvious. Keep them short and focused on qualification, not research findings.

## 3. Choose the study structure

Default structure:

1. Starting scenario.
2. One short warm-up or context question.
3. Two to four focused tasks.
4. A verbal response after each meaningful task.
5. Optional success, difficulty, rating, multiple choice, or written response blocks.
6. Short wrap-up.

Aim for 15-20 minutes for unmoderated tests. If the script is longer, recommend splitting it or cutting lower-priority questions.

## 4. Draft platform-realistic blocks

For each task, provide:

- UserTesting.com block type: task, verbal response, written response, multiple choice, rating scale, success metric, difficulty metric, Five Second Test, or comparison.
- Participant-facing wording.
- Research purpose.
- Bias check notes, if needed.

Task pattern:

```text
Task: [realistic user goal]
Verbal response: What was going through your mind as you did that?
Optional metric: Were you able to complete this task? / How difficult was this task?
Optional written response: one short structured answer only if needed.
```

## 5. Run the bias and length check

Check for:

- leading language;
- path-revealing instructions;
- tasks that test obedience instead of usability;
- too many tasks or questions;
- written responses that should be verbal;
- missing audience or screener logic;
- metrics that will not be used;
- comparison imbalance.

## 6. Prepare launch checklist

Before launch, tell the user to verify:

- participant count and audience segment count;
- screener logic;
- starting URL or prototype link access;
- device requirements;
- task order;
- response modes;
- expected duration;
- privacy or sensitive-data instructions;
- pilot test result.

Recommend piloting with one participant per audience segment when practical.

## Output format

Use this order:

1. Brief recap.
2. Audience and screener plan.
3. Test structure.
4. Script blocks.
5. Bias and length check.
6. Launch checklist.
7. Pilot recommendation.
